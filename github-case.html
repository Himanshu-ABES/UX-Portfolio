GitHub — Reducing merge friction

One-line: Redesigned collaboration and review flows to reduce friction, increase reviewer throughput, and improve first-time contributor success.

Role: Lead UX / Product Designer
Timeline: 3–4 months • Team: PM, 3 Eng, Researcher

Problem

Reviewers missed high-priority PRs due to noisy notifications and weak signals; first-time contributors struggled to meet merging requirements without additional hand-holding. This led to bottlenecks and uneven code velocity across teams.

Process

  • Stakeholder synthesis, analytics review and goal alignment to define success metrics.
  • Moderated usability tests (12 sessions) focused on PR discoverability and reviewer decision-making.
  • Sketches → wireframes → hi‑fi prototypes in Figma; motion specs for state transitions and microinteractions.
  • Implemented a feature-flagged A/B test with instrumentation for reviewer throughput and time-to-merge.

Design highlights

Introduced clearer reviewer cues (priority badges, compact summaries), inline action affordances (quick-approve, ask-for-change), and a lightweight prioritization surface so reviewers can triage faster without context switching.

Accessibility

Ensured color contrast for badges, keyboard-accessible keyboard flows for triage, and ARIA-friendly labels for dynamic list updates so assistive tech users receive the same signal fidelity.

Artifacts to include

  • Annotated before/after flow diagrams and decision trees
  • Figma prototypes showing microinteraction timing and motion specs
  • Experiment dashboards showing reviewer throughput, time-to-merge, and first-time contributor success

Outcome

Anonymized results: ~+18% reviewer throughput and ~12% reduction in average time-to-merge in the A/B cohort. Qualitative feedback indicated clearer prioritization and smoother onboarding for first-time contributors.

Metrics & measurement

  • Reviewer throughput: merged PRs per active reviewer per week (instrumented via code host telemetry).
  • Time-to-merge: distribution and median time between PR open and merge for controlled cohorts.
  • First-time contributor success: % of new contributors with a merged PR within 30 days.
